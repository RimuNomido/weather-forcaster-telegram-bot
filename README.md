# Weather Forecaster Telegram Bot

Погодный бот для Telegram с историей запросов. Показывает текущую погоду через GraphQL‑API Яндекса, запоминает ваши запросы и умеет их показывать или очищать.

## Возможности

- `/weather [город]` — погода сейчас. Без города — Тюмень.
- `/history` — последние 10 запросов с полными данными и общим числом сохранённых.
- `/history clear` — очистить свою историю.
- Все коды погоды переведены на русский, дополнены эмодзи (облачность, осадки, ветер, влажность).
- Каждый ответ API сохраняется в SQLite с привязкой к `user_id`.

## Как работает

1. Пользователь вводит город → бот получает координаты через Nominatim (OpenStreetMap).
2. Отправляет GraphQL‑запрос к Яндекс.Погоде.
3. Парсит ответ, превращает коды в текст и эмодзи.
4. Отдаёт результат и сохраняет сырой JSON в базу.

При `/history` достаются последние 10 записей, парсятся заново и выводятся одним сообщением.

## Установка и запуск

```bash
git clone https://github.com/RimuNomido/weather-forcaster-telegram-bot.git
cd weather-forcaster-telegram-bot
python -m venv venv
source venv/bin/activate   # или venv\Scripts\activate на Windows
pip install -r requirements.txt
```

Создайте в корне `.env`:

```
telegram_token=ваш_токен_бота
access_key=ваш_ключ_яндекс_погоды
```

- **Токен** — у [@BotFather](https://t.me/BotFather).  
- **Ключ** — зарегистрируйте приложение на [Яндекс.Погода](https://yandex.ru/dev/weather/), тариф «Тестовый» (бесплатно, 5000 запросов/сутки).

Запуск:

```bash
python bot.py
```

Бот отвечает в личных сообщениях.

## Ограничения

- Бесплатный тариф Яндекс.Погоды — 5000 запросов/сутки.
- Nominatim может быть медленным (для продакшена лучше кешировать координаты).
- История хранится вся, но `/history` показывает только последние 10.
